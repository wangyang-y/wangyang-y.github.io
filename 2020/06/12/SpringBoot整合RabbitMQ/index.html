1.4 编写配置类

使用@Coonfiguration标注这个类是一个配置类

在配置类中向容器中注入交换机组件Exchange，队列组件Queue、明确队列和交换机的绑定bind

package com.yang.springbootrabbitmq.config;

import org.springframework.amqp.core.*;
import org.springframework.beans.factory.annotation.Qualifier;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;


/**
 * @author WangYang
 * @version 1.0
 * @date 2020/6/16 20:47
 */
@Configuration
public class SpringBootRabbitMQ {
    public static final String EXCHANGE_NAME = "springboot-rabbitmq-exchange";
    public static final String QUEUE_NAME= "springboot-rabbitmq-queue";
    //注入交换机组件
    @Bean("bootExchange")
    public Exchange bootExchange(){
        //构建交换机,指定交换机类型和名字
        return ExchangeBuilder.topicExchange(EXCHANGE_NAME).durable(true).build();

    }

    //注入队列
    @Bean("bootQueue")
    public Queue bootQueue(){
        return QueueBuilder.durable(QUEUE_NAME).build();
    }

    //交换机和队列绑定
    @Bean
    public Binding bindQueueAndExchange(@Qualifier("bootQueue") Queue queue, @Qualifier("bootExchange") Exchange exchange){
        //指定哪个队列，哪个交换机，什么routingKey，参数啥的
        return BindingBuilder.bind(queue).to(exchange).with("boot.#").noargs();
    }
}

小结：

通常将队列名和交换机名，定义成静态常量方便在其他类中使用

创建交换机的时候就制定了交换机的类型ExchangeBuilder.topicExchange(EXCHANGE_NAME).durable(true).build();

构建队列QueueBuilder.durable(QUEUE_NAME).build();

绑定队列和交换机

BindingBuilder.bind(queue).to(exchange).with("boot.#").noargs();

1.4 编写消息监听器

@RabbitListener（queues=""） 监听queue的方法

Message 包含了所有的信息包括队列信息，以及配置的属性，和消息体

package com.yang.springbootrabbitmqconsumer.consumer;


import org.springframework.amqp.core.Message;
import org.springframework.amqp.rabbit.annotation.RabbitListener;
import org.springframework.stereotype.Component;

/**
 * @author WangYang
 * @version 1.0
 * @date 2020/6/16 21:22
 * 编写消息监听器
 */
@Component
public class MyListener {

    @RabbitListener(queues = "springboot-rabbitmq-queue")
    public void ListenerQueue(Message message){
        System.out.println(message);
        System.out.println(new String(message.getBody()));
    }
}
